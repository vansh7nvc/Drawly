## Components

### App

- **Purpose:** Main application container. Manages global state (tool, color, stroke, zoom, undo/redo stacks) and renders the toolbar and whiteboard.
- **Location:** `src/App.jsx`
- **Dependencies:** `react`, `fabric`, `Whiteboard.jsx`, `App.css`

### Whiteboard

- **Purpose:** Wrapper for the HTML5 Canvas via Fabric.js. Handles low-level events (pan, zoom interaction) and tool-specific configurations (brush settings).
- **Location:** `src/components/Whiteboard.jsx`
- **Dependencies:** `react`, `fabric`

## Data Flow

1. **Tool Selection:** User clicks a tool in `App.jsx` toolbar → `activeTool` state updates → `Whiteboard.jsx` re-configures Fabric canvas mode.
2. **Drawing:** User interacts with canvas → Fabric.js handles object creation/modification → `object:modified` etc. events trigger `saveState` in `App.jsx` → `undoStack` updates.
3. **Persistence:** `handleSave` stringifies canvas to JSON → saves to `localStorage`. `handleLoad` reads from `localStorage` → `canvas.loadFromJSON`.
